## 16.3.5 Типичные вопросы на собеседовании

### Теоретические вопросы

```
ТОПОВЫЕ ВОПРОСЫ JUNIOR PENTESTER (и ответы):
=============================================

Q1: "Объясните разницу между TCP и UDP"
    Ответ: TCP — надёжный, с установлением соединения
    (3-way handshake: SYN, SYN-ACK, ACK), гарантирует
    доставку. UDP — без соединения, быстрее но ненадёжный.
    В ИБ важно: SYN flood использует TCP handshake.

Q2: "Как работает SQL Injection?"
    Ответ: Пользовательский ввод попадает в SQL запрос
    без санитизации. Пример:
    SELECT * FROM users WHERE id='$id'
    Если $id = "1' OR '1'='1", то условие всегда истинно.
    Защита: Prepared Statements, Input Validation, WAF.

Q3: "Что такое XSS и какие типы бывают?"
    Ответ:
    • Reflected XSS — скрипт в URL, выполняется при клике
    • Stored XSS — скрипт сохранён в БД, выполняется у всех
    • DOM-based XSS — манипуляция DOM без запроса к серверу
    Опасность: кража cookies, session hijacking, keylogging.

Q4: "Объясните OWASP Top 10"
    Должны знать все 10 пунктов 2021 года:
    A01: Broken Access Control
    A02: Cryptographic Failures  
    A03: Injection
    A04: Insecure Design
    A05: Security Misconfiguration
    A06: Vulnerable and Outdated Components
    A07: Identification and Authentication Failures
    A08: Software and Data Integrity Failures
    A09: Security Logging Failures
    A10: SSRF

Q5: "Что такое SSRF?"
    Server-Side Request Forgery — заставить сервер делать
    запросы к внутренним ресурсам. Например:
    url=http://169.254.169.254/latest/meta-data/ (AWS metadata)
    Позволяет получить IAM credentials, внутренние сервисы.

Q6: "Разница между Authentication и Authorization?"
    Authentication: кто вы? (логин/пароль, MFA)
    Authorization: что вам разрешено? (роли, права)
    IDOR — нарушение Authorization (обход проверки прав).

Q7: "Как работает Pass-the-Hash?"
    Windows хранит NTLM хэши паролей. Если получить хэш
    (через Mimikatz, hashdump), можно использовать его
    для аутентификации БЕЗ знания пароля.
    psexec.py -hashes :NTLM_HASH domain/admin@target

Q8: "Что такое Kerberoasting?"
    В AD: сервисные аккаунты имеют SPN (Service Principal Name).
    Любой аутентифицированный пользователь может запросить
    TGS тикет. Тикет зашифрован хэшем пароля сервисного аккаунта.
    Атака: запросить тикет → взломать офлайн → получить пароль.
    GetUserSPNs.py domain/user:pass -request → hashcat

Q9: "Как бы вы провели внешний пентест?"
    Методология:
    1. Passive Recon: OSINT, Shodan, theHarvester
    2. Active Recon: nmap, nikto, gobuster
    3. Vulnerability Analysis: searchsploit, nessus
    4. Exploitation: Metasploit, manual exploits
    5. Post-Exploitation: persistence, lateral movement
    6. Reporting: документация, рекомендации

Q10: "Что такое CVSSv3 и как читать оценку?"
    CVSS = Common Vulnerability Scoring System
    Базовая оценка 0.0-10.0:
    Critical: 9.0-10.0 | High: 7.0-8.9
    Medium: 4.0-6.9  | Low: 0.1-3.9
    Факторы: Attack Vector, Complexity, Privileges Required,
    User Interaction, Confidentiality/Integrity/Availability Impact
```

### Практические вопросы и задания

```
ПРАКТИЧЕСКИЕ ЗАДАНИЯ НА СОБЕСЕДОВАНИИ:
=======================================

Задание 1: "Найдите уязвимость в этом коде"
```php
// Предложат что-то похожее:
$id = $_GET['id'];
$query = "SELECT * FROM products WHERE id = $id";
$result = mysqli_query($conn, $query);

// Вы должны:
// 1. Идентифицировать: SQL Injection
// 2. Объяснить как эксплуатировать
// 3. Предложить fix (prepared statements)
// 4. Объяснить какие данные можно получить
```

Задание 2: "Интерпретация nmap вывода"
```
PORT   STATE SERVICE VERSION
22/tcp open  ssh     OpenSSH 7.2p1
80/tcp open  http    Apache 2.4.7
443/tcp open  https
3306/tcp open  mysql  MySQL 5.5.47

Вопросы:
- Какие уязвимости можете предположить?
- С чего начнёте? (Apache 2.4.7 + MySQL 5.5.47 — старые версии)
- Что проверите на 80/443?
```

Задание 3: "CTF-подобное задание"
```
Часто дают небольшой CTF прямо на собеседовании:
- Простая SQL Injection форма
- XSS задание
- Найти скрытый флаг на тестовом сайте
- Декодировать base64 → caesar cipher → флаг

Совет: думайте вслух, объясняйте каждый шаг.
Интервьюер оценивает методологию, не только результат.
```

### Как подходить к техническому заданию

```

==================================

1. ДОКУМЕНТИРУЙ ВСЁ С ПЕРВОЙ МИНУТЫ
   mkdir task && cd task
   mkdir {recon,exploitation,screenshots,notes}
   # Запиши каждую команду в notes/commands.md

2. МЕТОДОЛОГИЧНОСТЬ ВАЖНЕЕ СКОРОСТИ
   Лучше найти 3 уязвимости методично, чем
   8 находок без структуры.

3. ПОКАЗЫВАЙ ДУМУ, А НЕ ТОЛЬКО РЕЗУЛЬТАТ
   В отчёте: "Я проверил X, результат Y, поэтому
   пошёл по вектору Z" — покажи процесс мышления.

4. КАЧЕСТВО ОТЧЁТА = 40% ОЦЕНКИ
   Оформляй как настоящий коммерческий отчёт.
   Используй шаблон из главы 16.2.

5. ЗАДАЙ УТОЧНЯЮЩИЕ ВОПРОСЫ ЗАРАНЕЕ
   "Можно ли использовать Metasploit?"
   "Включены ли социальная инженерия и фишинг?"
   "Какие IP/поддомены в scope?"
   
   Это показывает профессионализм, не незнание.

6. ЕСЛИ ЗАСТРЯЛ — НАПИШИ ОБ ЭТОМ
   "Пробовал вектор X, не сработало потому что Y.
   Переключился на Z." Мёртвые концы = реальный пентест.

7. PoC ДЛЯ КАЖДОЙ НАХОДКИ
   Скриншот или команда — доказательство каждой уязвимости.
   Без PoC — не уязвимость, а предположение.
