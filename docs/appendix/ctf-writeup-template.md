## Как пользоваться шаблоном

1. Скопируй нужный шаблон ниже
2. Заполни секции своими данными
3. Замени `[текст в скобках]` на реальные данные
4. Удали секции которые не применимы
5. Добавь скриншоты там где указано `[SCREENSHOT]`

## Советы по написанию хороших write-up

### Структура

```
ХОРОШИЙ WRITE-UP ВКЛЮЧАЕТ:

✅ Заголовок с категорией и очками
✅ Ссылки на источники
✅ Описание исходной точки (что видишь сразу)
✅ Описание процесса размышления (не только решение)
✅ Неудачные попытки (ценный опыт!)
✅ Объяснение payload — ПОЧЕМУ это работает
✅ Скриншоты / вывод команд
✅ Рекомендации по защите
✅ Использованные инструменты с версиями

❌ ПЛОХОЙ WRITE-UP:
- Только итоговый payload без объяснений
- "Я запустил sqlmap и получил флаг"
- Нет объяснения почему уязвимость существует
- Нет процесса решения (только ответ)
- Опубликован во время CTF (СПОЙЛЕР!)
```

### Примеры хорошего объяснения payload
<v-pre>
```

ПЛОХО:
"Я использовал payload: {{7*7}} и получил 49, значит SSTI"

ХОРОШО:
"Введя {{7*7}} в поле имени и получив ответ '49' (а не '{{7*7}}'),
я подтвердил что шаблонизатор интерпретирует выражения.
Результат '49' (а не '7*7' или ошибка) указывает на Jinja2/Twig.

В Jinja2 двойные фигурные скобки {{ }} обозначают выражение.
Выражение 7*7 вычисляется и результат (49) подставляется в HTML.
Это означает что я могу выполнять произвольный Python код через шаблонизатор.

Следующий payload для RCE:
{{''.__class__.__bases__[0].__subclasses__()[258](['cat','/flag.txt'],stdout=-1).communicate()[0]}}

Разбор:
- ''.__class__    → тип str
- .__bases__[0]  → базовый класс object
- .__subclasses__() → все подклассы (включая Popen)
- [258]          → subprocess.Popen (индекс может отличаться)
- (['cmd'],stdout=-1) → выполнить команду, захватить вывод
- .communicate()[0] → получить вывод"
</v-pre>
  
```

## Словарь CTF терминов

| Термин | Описание |
|--------|----------|
| **Flag** | Строка в формате CTF{...} — цель задания |
| **Writeup** | Технический разбор решения задания |
| **Payload** | Вредоносная нагрузка для эксплойта |
| **PoC** | Proof of Concept — демонстрация уязвимости |
| **Jeopardy** | Формат CTF: набор независимых заданий |
| **A/D** | Attack-Defense: атакуй чужих, защищай своих |
| **KotH** | King of the Hill: захват и удержание флага |
| **Pwn** | Успешная эксплуатация (взломать) |
| **Rev** | Reverse Engineering |
| **Chall** | Challenge (задание) |
| **OOB** | Out-of-Band (через сторонний канал) |
| **Blind** | Уязвимость без видимого вывода |
| **RCE** | Remote Code Execution |
| **LPE** | Local Privilege Escalation |
| **IDOR** | Insecure Direct Object Reference |
| **SSRF** | Server-Side Request Forgery |
| **SSTI** | Server-Side Template Injection |
| **XSS** | Cross-Site Scripting |
| **SQLi** | SQL Injection |
| **LFI** | Local File Inclusion |
| **Regex** | Regular Expression |
| **WAF** | Web Application Firewall |
| **Bypass** | Обход защитного механизма |
| **Enumerate** | Перечисление (поиск всех возможных значений) |
| **Fuzz** | Автоматизированный перебор |
| **Dork** | Специальный поисковый запрос |
| **IOC** | Indicator of Compromise |
| **OSINT** | Open Source Intelligence |
